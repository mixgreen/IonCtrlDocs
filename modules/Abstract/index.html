    Modules 简介

    0. 背景

    ARTIQ 提供了一整套操控底层硬件的接口，使得编排严密时序的操作成为可能。其指令是面向通道行为的形如：指定 ch1 在 3s 后打开，延迟 5s 后关闭；指定 ch2 的频率是100MHz，4s后打开，延迟 5s 后关闭。

    处在调试阶段 or 通道数比较少的情况下，这么做没问题。但通道数一多，这种单独指定每个通道行为的方式，就成为了实验人员的负担。实验人员在编写实验代码时的思考路径：我要先离子化（可能对应 ch1 打开），再冷却（可能对应 ch2、ch3 打开、ch1 关闭）。。。每个想执行的具体操作，都要经过一层转化，才能指向通道的行为。这十分麻烦。

    本 module 的第一个设计目的，就是简化麻烦。最终呈现的效果是，实验人员编写实验代码时，只需要关注想执行的行为（cooling、pumping、control、detecting、数据分析），而不必关注对通道行为的编排。这样拓展实验代码会十分方便。

    2. 注意事项

    关于 HasEnvirment：

    1. HasEnvironment 是用于封装实验模块的类，我们继承 HasEnvirment 封装了四个类：l369、l554、signal_mw、pmt。
    2. 在HasEnvirment 中，不要用 build，而是把 build 的内容全部塞进 _init_ 里, 并在主实验代码（继承 EnvExperiment 类，每隔实验代码都要有且只有一个 EnvExperiment 类）的 build 阶段实例化这个类。
    3. set_dataset 不能放进 HasEnvirment 的 __init__中，而是应该放在 prepare or build 中
    4. 当 run 被 kernel 装饰后，显式获得的数据需要转换成 artiq 支持的数据（假如这个数据要被传给传给 artiq 硬件的化）。传给 artiq 环境这一步会占用过多时间，因此最好把所有的数据获取和处理扔在同一个 rpc 函数里，不要传入 artiq。（以节省时间）
    5. 一个实验的数据集是共享的
    6. artiq 实验模块是不支持字典格式的
    7. 时序代码里如果要塞 print 这种非时序内容，就要提前封装一个异步执行的 print ，否则会导致时序报错
    8. 类嵌套也只能有两层，多于两层就找不到了（在config 里设置的参数导入时，要指定导入的参数组）
    9. 在创建 for 循环时，第一个循环用了2 s，后面的每个小循环用时是正常的
