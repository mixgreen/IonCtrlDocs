    management system

    Abstract

    artiq 的实验文件可以仅通过 artiq_run 来直接执行，不需要涉及 controller manager。 但面对实验类型怎多，数据处理流程复杂的情况，需要一个管理系统来管理实验的执行。

    管理系统的主要组件：

    • master：用于管理实验的 parameter 和 device database（在 build 中建立的），实验资源（在 repository 中），scheduling（执行实验的调度器），
    • controller manager：负责在计算机上运行和停止控制器

    ARTIQ client

    在 artiq_master（artiq_session） 运行后，本地可以在命令行运行 artiq_client 连接 master。

    artiq_client 的常见功能包括：

    1. 提交实验：示例 artiq_client submit repository/3_mw_example/Example_2_with_pmt.py 给出实验文件路径（项目根目录下的实验文件路径），submit 并 run。效果等同于 GUI 中的 submit。这个方法可以应用到云平台项目的执行脚本之中，以避免访问 GUI。
    2. 提交实验到不同的主机：artiq_client -s 192.168.1.100 --port 8888 submit my_experiment.py 如果 artiq_master 运行在别的主机上，可以指定该主机的域名和 master 执行的端口，来提交实验。用于限定实验的主控设备，并与之交互。感觉用处不大，主控设备跟所有的计算机都在同一个局域网下，任何一台机器都可以访问到 ARTIQ 硬件，也都可以自己单开一个 master。没必要专门连接到实验室电脑上。
    3. 重新扫描实验代码：artiq_client scan-repository 在改动了实验代码后（类名称、类注释、以及改 bug-以使得之前因 bug 而被跳过的代码），重新扫描实验文件。

    Scheduler

    Scheduler 维护一个实验队列，并提供一系列方法与实验队列交互

    worker_handlers.update({
    "scheduler_submit": scheduler.submit,
    "scheduler_delete": scheduler.delete,
    "scheduler_request_termination": scheduler.request_termination,
    "scheduler_get_status": scheduler.get_status,
    "scheduler_check_pause": scheduler.check_pause,
    "scheduler_check_termination": scheduler.check_termination,
    ...
})

    常见方法

    • submit: 提交一个新的实验到调度队列。
    • delete: 从调度队列中删除一个实验。
    • request_termination: 请求终止一个正在运行的实验。
    • get_status: 获取实验的当前状态或整个调度队列的状态。
    • check_pause: 检查一个实验是否被暂停。
    • check_termination: 检查一个实验是否被终止。
