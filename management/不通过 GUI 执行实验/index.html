    如何不通过 GUI 直接执行实验

    Abstract

    在 artiq_master （artiq_session）被执行后，主机与 core 仪器建立了连接。此后，我们通过 artiq_session 提供的 GUI 界面与 artiq_master 通信，完成实验流程调度、实验提交等。

    这个流程，对于本地调试的工程师而言，是完全足够的。不过，我们后期会涉及云平台项目，云端远程发送实验任务，本地接受这个任务之后执行。整个过程并不关心 GUI。

    因此，我们希望实现这样的功能：通过脚本文件 or jupyter notebook 执行实验程序，不通过 GUI。

    具体实现

    artiq_master 在执行后，可以访问 3251 端口的多个服务，包括：

    • master_config：配置文件

    • master_dataset_db：数据据数据库

    • master_device_db：仪器数据库

    • master_experiment_db：实验数据库

    • master_schedule：调度器，用于提交 - 执行实验 ，submit 方法就属于这个类

    我们可以通过 RPC 机制建立一个客户端，来访问这里的服务。示例代码如下：

    import artiq.frontend.artiq_client as artiq_client
import logging
import time

# 1. 建立连接
server = '::1'  # 指向本地主机
port = 3251  # 指明端口

scheduler = artiq_client.Client(server, port, "master_schedule")  # 获取调度器实例
dataset_db = artiq_client.Client(server, port, "master_dataset_db")  # 获取数据集实例
experiment_db = artiq_client.Client(server, port, "master_experiment_db")  # 实验文件数据库

# 2. 提交实验
# 描述想要提交的实验（文件、类名、日志等级、参数）
expid = dict(
    file="repository/Example_1_led_on_off_pulse.py",
    class_name="LED_On_Off_Pulse",
    log_level=logging.WARNING,
    arguments={}
    )


if not scheduler.get_status():
    # 如果调度器队列当前为空，就提交这个实验，否则先暂定
    rid = scheduler.submit(pipeline_name="main", expid=expid,
    priority=0, due_date=None, flush=False)
print(scheduler.get_status())

# 3. 在实验执行的过程中查询调度器的状态
while rid in scheduler.get_status():
    time.sleep(1.0)

    解读：

    1. artiq_client 封装了访问 master 服务的 python API，输入地址、端口、服务即可访问
    2. expid 封装了将要执行的实验信息
    3. scheduler.submit 用于提交实验，并返回实验的编号 rid 。检查 get_status 再执行，可以防止冲突。
